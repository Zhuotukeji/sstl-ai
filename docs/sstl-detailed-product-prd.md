# SSTL 搜索套利投放操作系统详细 PRD

## 1. 文档信息

- 文档版本：v1.0
- 适用范围：SSTL 竞品融合迭代，重点覆盖 P0/P1 产品需求
- 目标读者：老板、产品、设计、前端、后端、测试、投放运营负责人
- 竞品来源：AdRate、RSOC、SSTL 现有线上系统只读分析

## 2. 产品定位

SSTL 要从“投放配置 + 报表系统”升级为“搜索套利投放操作系统”。

核心不是增加几个菜单，而是形成完整闭环：

```mermaid
flowchart LR
  A["素材/文章/样式"] --> B["TikTok 批量建计划"]
  B --> C["内部 Campaign 投放链接"]
  C --> D["Offer 权重分发"]
  D --> E["上游收益回传"]
  E --> F["ROI/利润分析"]
  F --> G["自动化调控"]
  G --> B
```

## 4. 全局设计原则

### 4.1 设计风格

- 整体应是运营后台风格：信息密度高、表格清晰、筛选强、动作明确。
- 避免营销式大卡片和大面积装饰图；首屏优先展示待处理风险、今日消耗、收益、利润、ROI。
- 页面以“筛选区 + 指标条 + 数据表 + 右侧抽屉详情”为主要结构。
- 操作按钮使用稳定位置：主操作在页面右上，行内操作在表格右侧固定列。
- 危险动作必须二次确认，批量危险动作需要展示影响对象数量。

### 4.2 交互规范

- 列表页筛选项支持“查询、重置、保存视图”。
- 表格支持列设置、密度切换、横向滚动、固定首列与操作列。
- 所有 ID、投放链接、落地页 URL 支持一键复制。
- 所有金额默认美元，保留 2 位小数；ROI/CTR/CVR 等比例保留 2 位。
- 状态用颜色区分：
  - 正常/启用：绿色
  - 暂停/停用：灰色
  - 风险/待处理：橙色
  - 失败/拒绝/异常：红色
  - 增强展示/自动化：紫色
- 空态必须给出下一步动作，例如“暂无 Offer，立即创建 Offer”。
- 加载态使用 skeleton 或表格 loading；不能出现整页白屏。
- 错误态展示错误原因、重试按钮和错误 ID。

## 6. 核心页面需求

## 6.1 控制台

### 页面目标

让管理者和优化师一进系统就知道今天是否赚钱、哪里亏、哪里需要处理。

### 首屏模块

- 今日核心指标：消耗、收益、利润、ROI、CPA、RPM、异常跳转数。
- 风险待办：
  - ROI 低于阈值 Campaign
  - 高消耗无转化 Adset
  - TONIC declined 对象
  - Offer 无可用落地页
  - 自动化执行失败
- 趋势图：近 7/14/30 天 spend、revenue、profit、ROI。
- Top/Bottom 榜：
  - Top 10 高利润 Campaign
  - Bottom 10 亏损 Campaign
  - Top 10 素材
  - 异常跳转最多 Campaign

### 交互

- 指标卡点击进入对应报表，并自动带上日期和筛选条件。
- 风险待办点击打开右侧抽屉，展示对象详情和建议动作。
- 支持按优化师、业务标签、媒体账号筛选全页数据。

### 设计优化

- 指标卡保持紧凑，不做大 Hero。
- 利润为负时整行风险提示变红，但不要大面积红底。
- 图表最多同时展示 2-3 条线，避免噪音。

## 6.2 Offer 管理

### 页面目标

管理所有可分发的上游 Offer，明确每个 Offer 的平台、落地页、Pixel 注入、绑定 Campaign 和可用状态。

### 列表字段

| 字段 | 说明 |
| --- | --- |
| Offer 名称 | 支持点击进入详情 |
| 平台 | TONIC/System1/自定义 |
| 落地页 URL | 支持复制，超长省略 |
| 外部 Campaign ID | 上游平台 campaign id |
| Pixel 注入 | 启用/未启用 |
| Pixel | Pixel 名称或 ID |
| 业务标签 | 彩色标签 |
| 绑定 Campaign 数 | 点击查看绑定列表 |
| 状态 | active/inactive |
| 更新时间 | 最后编辑时间 |
| 操作 | 编辑、启停、查看绑定 |

### 筛选项

- 关键词：Offer 名称/落地页/外部 Campaign ID
- 平台
- 业务标签
- Pixel 注入状态
- Offer 状态

### 新增/编辑交互

使用右侧抽屉，不跳页。

字段：

- Offer 名称，必填，最长 100 字符
- 平台，必填
- 落地页 URL，必填，必须是合法 URL
- 外部 Campaign ID，选填
- 是否注入 Pixel，默认否
- Pixel，注入时必填
- Pixel 参数名，默认读取平台配置
- 业务标签，选填
- 备注，选填
- 状态，默认启用

### 状态与校验

- 停用 Offer 后，不再参与权重分发。
- 如果 Offer 已绑定启用 Campaign，停用时二次确认并提示影响 Campaign 数。
- 落地页 URL 改动时记录操作审计。

### 空态

文案：暂无 Offer。创建 Offer 后即可绑定到内部 Campaign 做权重分发。  
按钮：创建 Offer

### 页面目标

统一管理搜索套利投放链接，连接 TikTok 广告和上游 Offer 分发。

### 列表字段

| 字段 | 说明 |

| 投放链接 | `https://domain/r/{campaign_id}`，支持复制 |
| 投放域名 | 绑定域名 |
| 业务标签 | 彩色标签 |
| Offer 数 | 当前绑定 Offer 数 |
| 权重状态 | 正常/无 Offer/含停用 Offer |
| 反跟踪保护 | 启用/未启用 |
| 增强展示 | 启用/未启用 |
| 状态 | active/inactive |
| 创建时间 | 创建时间 |
| 操作 | 编辑、权重配置、复制链接、启停 |

### 筛选项

- Campaign 名称
- Campaign ID
- 投放域名
- 业务标签
- 状态
- 是否启用反跟踪
- 是否启用增强展示

### 创建流程

采用三步抽屉或分步弹窗：

1. 基础信息
   - Campaign 名称
   - Campaign ID，默认自动生成，可手动改
   - 投放域名
   - 业务标签
   - 描述
2. 分发与风控
   - 必需参数，默认 `ttclid`、`campaign_id`、`ad_id`
   - TikTok macro 参数
   - 目标国家
   - 兜底页或兜底 URL
   - 反跟踪保护
   - 增强展示
3. Offer 权重
   - 添加 Offer
   - 设置权重
   - 展示权重占比
   - 校验至少 1 个启用 Offer

### 权重配置交互

- 使用弹窗或抽屉表格。
- 列：Offer、平台、状态、权重、占比、操作。
- 权重输入为正整数，总和不要求 100。
- 占比实时计算。
- 停用 Offer 灰显，并提示“不参与分发”。
- 保存前展示影响说明：“保存后新访问将按新权重分发，历史日志不受影响。”

### Campaign 详情

详情页使用 Tabs：

- 基本信息
- Offer 权重
- 重定向日志
- 报表趋势
- 操作记录

### 关键状态

- 正常：启用且至少 1 个启用 Offer。
- 无可用 Offer：Campaign 启用但无启用 Offer，列表显示红色风险。
- 参数缺失风险：未配置必需参数，显示橙色提示。
- 域名缺失：无法生成投放链接，禁止启用。

## 6.4 公开跳转链路

### 页面目标

这个不是后台页面，但必须定义访问逻辑。

### 访问入口

`GET /r/{campaign_id}`

### 处理顺序

1. 生成 `hit_id`。
2. 获取或生成 `visitor_id`。
3. 查询 Campaign。
4. 校验 Campaign 状态。
5. 校验必需参数。
6. 校验国家。
7. 获取启用 Offer。
8. 按权重选择 Offer。
9. 拼接参数并跳转。
10. 写 Redirect Log。

### 跳转类型

- `normal`：正常跳转 Offer。
- `fallback`：走兜底页。
- `blocked`：被反跟踪或规则拦截。
- `enhanced`：进入增强展示后跳转。

### 设计要求

- 跳转链路必须快，配置读取需要缓存。
- 日志写入不能阻塞主跳转，可异步写入。
- URL 参数中的敏感值需要脱敏后入库。

## 6.7 自动化计划

### 页面目标

把现有自动关停升级为可配置、可预览、可追溯的自动化运营系统。

### 列表字段

| 字段 | 说明 |
| --- | --- |
| 计划名称 | 点击查看详情 |
| 规则维度 | 广告账户/Campaign/Adset/内部 Campaign/业务标签 |
| 触发方式 | 固定时间/间隔执行/数据同步后 |
| 时区 | 默认 Asia/Shanghai |
| 冷却时间 | cooldown_minutes |
| 规则数 | 绑定规则数量 |
| 启用状态 | enabled |
| 最近执行 | 最近一次执行时间 |
| 操作 | 编辑、复制、dry-run、启停、删除 |

### 创建流程

1. 选择目标维度
   - 广告账户
   - TikTok Campaign
   - Adset
   - 内部 Campaign
   - 业务标签
2. 配置触发方式
   - 固定时间：每天一个或多个时间点
   - 间隔执行：5/15/30/60 分钟
   - 数据同步后执行
3. 配置规则
   - 条件组
   - 动作组
   - 冷却时间
4. Dry-run 预览
   - 命中对象
   - 命中原因
   - 拟执行动作
5. 保存并选择是否启用

### 规则条件

广告侧：

- spend
- cash_spend
- CPA
- CPC
- CPM
- CTR
- clicks
- impressions
- conversions
- budget
- bid

收益侧：

- revenue
- profit
- ROI
- profit_rate
- RPM
- RPA
- critical CPA

素材侧：

- video views
- 2s/6s views
- 25/50/75/100% views
- material ROI

文本：

- Campaign 名称包含/不包含
- Adgroup 名称包含/不包含
- 关键词包含/不包含

### 动作

- 通知：邮件、钉钉、飞书/Lark。
- 广告动作：启用、暂停、调预算、调出价、调转化出价。
- 分发动作：Offer 降权、Offer 暂停、Campaign 切兜底、启用增强展示。

### 设计优化

- 规则编辑器采用“如果 / 且 / 或 / 则”的自然语言结构。
- 条件组支持折叠，避免长规则难读。
- 右侧实时显示规则摘要，例如：
  - 如果最近 1 天 spend > 50 且 ROI < 1，则暂停 Campaign 并通知钉钉。
- Dry-run 结果必须在启用前展示。

## 6.9 TikTok 跨账号复制

### 页面目标

吸收 AdRate 的复制体验，减少投放人员重复建计划成本。

### 流程

1. 选择源授权/源广告账户。
2. 选择源 Campaign。
3. 选择复制范围：Campaign、Adgroup、Ad、素材、CTA、Pixel、落地页。
4. 选择目标广告账户。
5. 配置覆盖项：名称后缀、预算、出价、Pixel、落地页、排期。
6. 预览任务。
7. 提交任务。
8. 在任务中心查看结果。

### 交互优化

- 源 Campaign 支持搜索和按状态筛选。
- 目标账号支持批量选择，并显示账号余额/授权状态。
- 不支持复制的对象要提前标出原因。
- 提交前展示预计创建 Campaign/Adgroup/Ad 数量。

- declined 时暂停 Offer
- declined 时 Campaign 切兜底
- recovered 时生成恢复预览，人工确认

### 设计优化

- declined 不能只显示红色状态，要给“建议处理动作”。
- 自动联动默认关闭，必须管理员开启。
- 开启前必须展示影响范围。

## 7. 报表指标统一口径

| 指标 | 公式 | 说明 |
| --- | --- | --- |
| spend | TikTok 消耗 | 广告成本 |
| revenue | 上游收益 | TONIC/System1 等 |
| profit | revenue - spend | 利润 |
| ROI | revenue / spend | spend 为 0 时为 0 |
| profit_rate | profit / revenue | revenue 为 0 时为 0 |
| RPM | revenue / impressions * 1000 | 千次展示收益 |
| RPA | revenue / conversions | 单转化收益 |
| critical CPA | revenue / conversions | 临界 CPA |
| CPA | spend / conversions | 单转化成本 |
| CTR | clicks / impressions | 点击率 |

## 9. 通用状态设计

### 9.1 列表空态

- 无数据：说明模块价值 + 主操作按钮。
- 有筛选无结果：提示“没有符合条件的数据”，提供“清空筛选”。

### 9.2 表单错误

- 必填项错误在字段下方展示。
- 业务校验错误在抽屉顶部展示，例如“当前 Campaign 无启用 Offer，不能启用”。

### 9.3 异步任务状态

- 提交任务后不阻塞页面，跳转任务中心。
- 右上角显示任务提交成功 toast。
- 任务失败时在任务中心展示失败明细，不只弹 toast。

### 9.4 危险动作确认

确认弹窗必须包含：

- 动作名称
- 影响对象数量
- 可能后果
- 确认按钮文案，例如“确认暂停 12 个 Campaign”

## 10. 设计优化清单

- 把“配置类页面”和“运营动作页面”分开，避免投放人员在深层配置里找日常操作。
- 旧报表过多时，用“数据洞察”聚合入口承接，不一次性删除旧入口。
- 关键对象统一详情抽屉：Campaign、Offer、素材、任务、规则。
- 所有列表提供保存视图，投放团队常用筛选可以复用。
- 所有表格关键列固定，避免横向滚动时丢失上下文。
- 对 ROI、profit、CPA 使用颜色强调，但颜色不要过多。
- 批量操作前先做预览，减少误操作。
- 自动化规则必须自然语言摘要，降低配置理解成本。
- 日志类页面默认按时间倒序，保留最近 30 天提醒，长期数据通过导出或归档查看。
- 对外部平台凭证、Pixel、token 等敏感信息只展示脱敏值。

## 11. P0 上线验收

- 可以创建 Offer，并在列表中筛选、启停。
- 可以创建内部 Campaign，并生成投放链接。
- 可以给 Campaign 绑定多个 Offer 并设置权重。
- 访问投放链接后可以按权重跳转到 Offer。
- 参数异常、Campaign 停用、无 Offer 时进入 fallback 或 blocked。
- 每次访问生成重定向日志。
- 重定向日志支持筛选、详情、复制排查信息。
- Campaign 报表展示 spend、revenue、profit、ROI。
- 自动化 dry-run 可以展示命中对象和拟执行动作。
- 所有写操作有操作审计。

## 12. P1 上线验收

- 跨账号复制支持源账号、源 Campaign、目标账号、覆盖配置、任务预览。
- 任务中心支持进度、失败明细、重试。
- 素材 AI 分析支持 AI score、风险标签、建议用途。
- TONIC 合规报告可同步并展示 declined 对象。
- 合规 declined 可进入自动化 dry-run。

## 13. 非目标

- V1 不做 SaaS 订阅、付费套餐和额度商业化。
- V1 不一次性重构所有旧报表。
- V1 不默认开启自动化真实执行。
- V1 不做复杂 AI 自动投放决策，只做人机协同建议。
